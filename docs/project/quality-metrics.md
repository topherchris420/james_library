# Quality Metrics

This document defines reliability and maintainability metrics that are tracked in CI and reviewed monthly.

## Scope and cadence

- **Scope:** `src/**` production paths and CI test artifacts generated by the quality report workflow.
- **Cadence:** automated monthly report + maintainers monthly review.
- **Primary output:** `quality-metrics-report.md` and `quality-metrics-report.json` CI artifacts.

## Metric definitions, baselines, and targets

| Metric | Definition | Baseline (2026-03) | Target (2026-09) | Threshold policy |
|---|---|---:|---:|---|
| Panic count in production path | Count of `panic!(` matches under `src/**` excluding test-only paths | 28 | <= 10 | Must trend downward month-over-month; any increase requires explanation in PR quality delta |
| Unwrap count in production path | Count of `.unwrap(` matches under `src/**` excluding test-only paths | 2800 | <= 2000 | Must trend downward quarter-over-quarter |
| Flaky test rate | `%` of tests that fail in run 1 and pass in immediate rerun (`run1.xml` vs `run2.xml`) | 2.0% | <= 0.5% | >1.0% triggers maintainer triage ticket |
| Mean PR size | Mean changed lines (adds + deletes) across last 30 days of merge history | 650 LOC | <= 400 LOC | >500 LOC mean triggers split-PR recommendation |
| Critical-path test coverage | `%` of known critical-path tests discovered and executed in report run | 82% | >= 95% | <90% blocks quality status from being considered healthy |

## Data sources

- **Static scan:** ripgrep counts for `panic!(` and `.unwrap(` in production paths.
- **Test artifacts:** JUnit XML from two consecutive test runs for flaky rate.
- **Critical-path coverage:** `cargo nextest list` test inventory + JUnit execution records.
- **PR size:** `git log --since=30.days --numstat` line delta mean.

## Notes

- Baselines are bootstrap values to establish trend tracking, not acceptance of current risk.
- If repository layout or test harness changes, update this document and `scripts/ci/quality_metrics_report.py` in the same PR.
